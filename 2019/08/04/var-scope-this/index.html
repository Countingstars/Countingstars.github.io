变量 作用域 this

发表于 2019-08-04 更新于 2019-08-06 分类于 JS 阅读次数：

基本类型和引用数据类型

ES变量可能包含两种不同的数据类型的值：基本类型值和引用类型值。

基本数据类型：目前共有Undefined、Null、Boolean、Number、String、BigInt、Symbol这7中类型，基本数据类型变量在stack（栈内存）中存储标识符与值。基本类型可以直接操作保存在变量中的实际的值，所以是按值访问的。

引用类型：是所有Object类型。引用类型变量在Stack（栈内存）中存储标识符和指向Heap(堆内存)中值的地址,在堆内存中存放具体值。（可动态添加删除属性）

变量复制和参数传递

JavaScript变量的拷贝都是按栈内存内的值传递， 函数的参数传递与变量复制传递表现一致

检测类型

typeof操作符，是确定基本数据类型的最佳变量

instanceof,如果变量是给定引用类型的实例（根据构造函数的prototype属性是否出现在对象的原型链中的任何位置来判断），则返回true

作用域

作用域为一套规则，这套规则用来管理引擎如何在当前作用域以及嵌套的子作用域中根据标识符名称进行变量查找。Js采用的是词法作用域，词法作用域是由你在写代码时将变量和块作用域写在哪里来决定的。

气泡1包含着整个全局作用域，其中只有一个标识符foo；

气泡2包含着foo()创建的作用域，其中有三个标识符，b，a,bar;

气泡3包含这个bar()创建的作用域，其中只有一个标识符，c

引擎在执行console时，会查找a,b,c三个变量，首先从最内部的bar开始查找，如果无法在改作用域找到a,b,会去上一级嵌套的foo查找，这样逐级向上查找，直到找到第一个匹配的标识符时停止

下面是引擎查找匹配标识符时涉及到的一些概念：

  1. 执行环境（execution context），定义了变量和函数有权访问的其他数据。

  2. 每个执行环境有一个与之关联的变量对象（variable object），环境中定义的所有变量和函数都保存在这个对象中。

  3. 每个函数都有自己的执行环境，当执行流进入一个函数时，函数的环境会被推入一个环境栈中，在函数执行之后，栈将其环境弹出，把控制权返回给之前的执行环境。

  4. 作用域链（scope chain），保证对执行环境有权访问的所有变量和函数的有序访问,实际是一个指向变量对象的活动列表

闭包是有权访问另一个函数作用域的中变量的函数

this

引入this的初衷就是想在原型继承的情况下，拿到函数的调用者

上面那句话是让我恍然大悟一下子明白this绑定谁，this只看函数执行时的调用方式，不管在哪里声明。接下来详细讲下绑定this的四种规则

默认绑定

foo()作为独立函数调用，因此使用默认绑定，this指向全局对象，注意，在严格模式下，全局对象将无法使用默认绑定，this会绑定到undefined上

隐式绑定

当函数引用有上下文对象时，隐式绑定规则会把函数调用中的函数的this绑定到这个上下文对象上

隐式丢失

虽然fn是一个obj.foo的引用，但实际上引用的是foo函数本身，fn此时其实是一个独立函数调用，应用默认绑定。所以函数参数传递，类似的回调函数调用，都会丢失this绑定，在非严格模式下，默认绑定全局，在DOM文档模式中即为window

显式绑定

JavaScript提供的绝大多数函数以及你自己创建的所有函数都可以使用call(..)和apply(..)方法，在调用函数时绑定this

fuc.apply(thisArg, argsArray)

apply()接收两个参数，一个是一个对象，将this强制绑定到这个obj上， 一个是数组或类数组对象，其中的数组元素将作为单独的参数传给func。

func.call(thisArg, arg1, arg2, arg3…)

call()方法和apply()方法作用相同，区别仅在接收参数的方式不同，调用call()方法时，传递给函数的参数必须逐个列举出来

bind

这种硬绑定是一种非常常用的模式，所以ES5提供了内置方法Function.prototype.bind，bind()方法创建一个新函数，这个新函数的this被bind的第一个参数指定，其余参数将作为新函数的参数供调用时使用。

第三方库的许多函数，以及JavaScript语言和宿主环境中许多新的内置函数，都提供了一个可选的参数，通常被称为“上下文”（context），其作用和bind(..)一样，确保你的回调函数使用指定的this。

new绑定

使用new来调用函数，即发生构造函数调用时，会自动执行以下操作:

  1. 创建（或者说构造）一个全新的对象。

  2. 这个新对象会被执行[[原型]]连接。

  3. 这个新对象会绑定到函数调用的this。

  4. 如果函数没有返回其他对象，那么new表达式中的函数调用会自动返回这个新对象

所以使用new来调用foo时，会构造一个新对象并把它绑定到foo(..)调用中的this上

总结：判断函数中this绑定，首先需要找到函数的直接调用位置，然后按顺序应用以下四种规则

  1. 由new调用？绑定新创建的对象上
  2. 由call或者apply或者bind调用？绑定到指定的对象上
  3. 由上下文调用？绑定到上下文对象上
  4. 默认：严格模式下绑定undefined，否则绑定到全局对象上
  5. ES6的箭头函数不会应用上述四种规则，而是继承外层函数调用的this绑定。
